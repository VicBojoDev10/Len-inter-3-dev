### 1. ¿Qué es Visual Studio Code?
Es un software que nos sirve para programar codigo utilizando diferentes extensiones para poder hacerlo mas sencillo. 
### 2. ¿Para qué sirve Visual Studio Code?
Visual Studio Code nos sirve para poder crear codigo que puede ser extendido a git y así hacer nuestro trabajo más remoto así como poder unirle extensiones para poder hacerlo visible en formato web, ponerle otros idiomas, un servidor interno, entre muchas otras opciones.
### 3. ¿Qué es la Terminal de Comandos?
La cmd es un programa que nos sirve para poder ingresar codigo que nos pueda ayudar tanto a modificar, añadir, eliminar, examinar, entre muchas otras cosas. 
### 4. ¿Para qué sirve la Terminal de Comandos?
Generalmente sirve para administrar nuestra computadora y los usuarios pero igual en VSC sirve para poder navegar en el programa.
### 5. ¿Qué es Markdown?
Markdown es una extensión de html que hace mas sencillo el proceso de codigo en una pagina web y suele ser muy utilizado.
### 6. ¿Para qué sirve Markdown?
Markdown sirve para poder hacer formatos, meter imagenes, agregar texto, poner negritas, cursiva, encabezados, etc.
Para hacerlo mucho mas sencillo y que tenga formato para pagina web.
### 7. ¿Qué es Git?
Git es una plataforma que sirve para subir archivos y poder almacenarlos en repositorios para que sirvan como servidor y para implementar cambios conforme lo requiera el usuario.
### 8. ¿Para qué sirve Git?
Git nos sirve para crear, subir, eliminar, mover, duplicar archivos y que podamos usar el mismo codigo para subirlos a un servidor que nosotros dediquemos para esos archivos.
### 9. ¿Qué es GitHub?
GitHub es una plataforma que utiliza de base git pero que su enfoque es mas a la comunidad y sirve como red social en si.
### 10. ¿Para qué sirve GitHub?
Sirve para crear repositorios y poder compartirlos con la comunidad que se desee sobre todo sirve mucho para trabajos colaborativos de developer y suele utilizarse mucho para compartir codigo.
### 11. ¿Qué es Git Bash?
GitBash es una extensión para poder utilizar git y github de forma local y que se puedan subir y bajar archivos del repositorio deseado usando codigo de git y una cuenta de github para hacer cambios desde una consola.
### 12. ¿Para qué sirven los siguientes comandos: pwd, whoami, touch, mkdir, cp, mv, ls, clear, cd y rm?. Explica su función y escribe un ejemplo de su sintaxis, para ello usa bloques de código en formato markdown.

Pwd: Para mostrar el directorio actual.
```bash
$ pwd
/d/Entregas-practicas.
```
Whoami: El usuario que está en la cuenta.
```bash
$ whoami
Morado15
```
mkdir: Crear carpeta.
```bash
$ mkdir prueba
```
cp: Copiar un archivo y moverlo a una carpeta
```bash
$ cp image.png prueba/image.png
```
mv: Mover un archivo.
```bash
$ mv prueba/image.png prueba2/image.png
```
ls: Lista de archivos.
```bash
$ ls 
image.png  Imagenes/  LICENSE  Practica-1.md  practica-5.md  prueba/  prueba2/  README.md
```
clear: Limpiar la terminal.
```bash
$ clear
```
cd: Cambio de directorio.
```bash
$ cd D://build
```
rm: Eliminar un archivo.
```bash
$ rm prueba
```
### 13. ¿Qué significan los siguiente caracteres en la terminal de Comandos: ./, ../, /, y ~?

./= Para buscar un archivo en la carpeta actual.

../= Para buscar un archivo en la carpeta que está mas arriba.

/= Es el directorio Raiz o Root.

~=Representa el directorio Home.
